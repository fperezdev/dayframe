# DayFrame

A full-stack, offline-first mobile task scheduler built with **Expo (React Native)** + **NestJS**.

## Stack

| Layer | Technology |
|---|---|
| Mobile | Expo (React Native) + TypeScript + Expo Router |
| UI | React Native Paper (Material Design 3) |
| Local DB | SQLite via `expo-sqlite` |
| State | Zustand |
| Forms | react-hook-form + zod |
| Backend | NestJS + TypeScript |
| Cloud DB | PostgreSQL on [Neon](https://neon.tech) |
| Auth | JWT (access 15 min + refresh 30 days) + bcrypt |
| Push | Expo Push Service |
| Hosting | [Render](https://render.com) (free tier) |

## Monorepo structure

```
dayframe/
├── apps/
│   ├── mobile/   # Expo app
│   └── api/      # NestJS backend
└── packages/
    └── shared/   # Shared TypeScript types & constants
```

## Prerequisites

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (for the API + Postgres)
- Node.js 20+ and npm 10+ (for the Expo mobile app on the host)
- [EAS CLI](https://docs.expo.dev/eas/) — `npm i -g eas-cli` (for CI builds)

## Local development

The API and Postgres run inside Docker. The Expo mobile app runs on your host
because Metro needs a direct network connection from your phone or emulator.

### 1. Configure environment

```bash
cp .env.example .env
```

Edit `.env` and set at minimum:

| Variable | Value for Docker dev |
|---|---|
| `DATABASE_URL` | Already set correctly — points at the Docker Postgres container |
| `JWT_SECRET` | Any random string |
| `JWT_REFRESH_SECRET` | Any random string |
| `EXPO_PUBLIC_API_URL` | Your machine's **LAN IP** + port 3000, e.g. `http://192.168.1.100:3000` (so physical devices/emulators can reach the API container) |

> `localhost:3000` works when running the Expo simulator on the same machine
> but will fail on a physical device. Use `ipconfig` (Windows) or
> `ifconfig` (macOS/Linux) to find your LAN IP.

### 2. Start the API and Postgres

```bash
# From monorepo root
docker compose up --build
```

This will:
1. Start a Postgres 16 container and wait until it is healthy.
2. Build the API image (installs deps, generates Prisma client).
3. Run `prisma migrate deploy` inside the container on first start.
4. Start NestJS in watch mode — file changes in `apps/api/` and `packages/shared/` hot-reload automatically.

Subsequent starts (no rebuild needed):

```bash
docker compose up
```

Stop everything:

```bash
docker compose down          # keeps postgres data volume
docker compose down -v       # also removes the postgres data volume (full reset)
```

### 3. Start the mobile app

In a separate terminal, from the monorepo root:

```bash
npm install        # first time only
npm run mobile
```

Scan the QR code with the **Expo Go** app on your phone, or press `a`/`i` for
Android/iOS emulator.

### Useful commands while running

```bash
# View API logs
docker compose logs -f api

# Open a shell inside the API container (e.g. to run prisma studio)
docker compose exec api sh
npx prisma studio --schema=apps/api/prisma/schema.prisma

# Create a new migration after editing schema.prisma
docker compose exec api sh -c \
  "npx prisma migrate dev --name <migration-name> --schema=apps/api/prisma/schema.prisma"
```

## Deployment

Deployments are fully automated via the **GitHub Actions workflow** at
`.github/workflows/deploy.yml`. Every push to `main`:

1. Runs lint and type-check across all packages.
2. Triggers a **Render** deploy for the API (on success).
3. Kicks off an **EAS preview build** for iOS and Android (on success).

### Required GitHub secrets

Go to **Settings → Secrets and variables → Actions** in your GitHub repo and
add the following secrets:

| Secret | How to get it |
|---|---|
| `RENDER_DEPLOY_HOOK_URL` | Render dashboard → your service → **Settings** → **Deploy Hook** → copy the URL |
| `EXPO_TOKEN` | [expo.dev](https://expo.dev) → Account settings → **Access tokens** → create a token |

### One-time setup

#### API → Render

1. Push the repo to GitHub.
2. Go to [render.com](https://render.com) → **New → Web Service** → connect your repo.
3. Render detects `render.yaml` and pre-fills build/start commands.
4. Under **Environment**, set `DATABASE_URL` to your Neon connection string (all other vars are auto-generated by `render.yaml`).
5. After the first manual deploy succeeds, copy the **Deploy Hook URL** from **Settings** and save it as the `RENDER_DEPLOY_HOOK_URL` GitHub secret.

#### Mobile → EAS

1. Install EAS CLI: `npm i -g eas-cli`
2. Log in: `eas login`
3. Inside `apps/mobile`, link the project to your Expo account: `eas build:configure`
   This writes an `extra.eas.projectId` into `app.json`.
4. Create an [Expo access token](https://expo.dev/accounts/[account]/settings/access-tokens) and save it as the `EXPO_TOKEN` GitHub secret.
5. From that point on, every push to `main` will trigger an EAS preview build automatically.

#### Database migrations on deploy

Render runs `npx prisma generate` as part of the build command (see `render.yaml`).
For **schema migrations** (`prisma migrate deploy`), run them manually from your
local machine against the production `DATABASE_URL` before or after a deploy:

```bash
DATABASE_URL="<production-url>" npx prisma migrate deploy --schema=apps/api/prisma/schema.prisma
```

## Key features

- **Offline-first** — all data is stored locally in SQLite; syncs to Postgres when online
- **Task CRUD** — title, description, priority, status, scheduled date, due date, and recurrence
- **Today screen** — date header, completion progress bar, and pending/completed filter chips
- **Calendar view** — monthly calendar with dot markers on days that have tasks
- **All Tasks screen** — text search with status, priority, and category filters
- **Categories** — color-coded tags, manageable from the Settings screen
- **Reminders** — configurable frequency and time window, scheduled as local notifications
- **Recurring tasks** — RRULE-based (None, Daily, Weekdays, Weekly, Monthly)
- **Cloud sync** — last-write-wins, bidirectional sync via `/sync` endpoint
- **Push notifications** — Expo Push Service; server cron sends reminders every 30 minutes
- **Dark mode** — automatic light/dark theme via Material Design 3

## Available scripts

Run from the monorepo root:

| Script | Description |
|---|---|
| `npm run dev` | Start both API (watch mode) and Expo dev server in parallel |
| `npm run mobile` | Start Expo dev server only |
| `npm run api` | Start NestJS in watch mode only |
| `npm run build` | Build all packages and apps |
| `npm run lint` | Lint all packages |
| `npm run test` | Run all tests |
| `npm run type-check` | Type-check all packages |

## API endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/auth/register` | Public | Create account |
| POST | `/auth/login` | Public | Login |
| POST | `/auth/refresh` | Public | Refresh access token |
| POST | `/auth/push-token` | JWT (user) | Save Expo push token |
| POST | `/auth/agent-token` | JWT (user) | Generate a short-lived agent token |
| GET | `/tasks` | JWT (user) | List tasks |
| POST | `/tasks` | JWT (user or agent) | Create task |
| PATCH | `/tasks/:id` | JWT (user) | Update task |
| DELETE | `/tasks/:id` | JWT (user) | Soft-delete task |
| GET | `/categories` | JWT (user) | List categories |
| POST | `/categories` | JWT (user) | Create category |
| PATCH | `/categories/:id` | JWT (user) | Update category |
| DELETE | `/categories/:id` | JWT (user) | Soft-delete category |
| POST | `/sync` | JWT (user) | Bidirectional sync |
